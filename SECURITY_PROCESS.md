# 🛡️ Skills 安全安装流程

## 流程概述

本 Skill 整合了完整的 Skills 安装安全检查流程，在安装任何 Skill 前自动执行双层验证。

## 判定标准详解

### 第一层：ClawHub 评分检查

**执行命令**: `clawhub search <skill-name>`

**评分阈值**: 3.5 分

| 评分范围 | 判定 | 操作 | 说明 |
|---------|------|------|------|
| **≥ 3.5** | ✅ 高评分 | 进入沙箱扫描 | 社区认可的高质量 Skill |
| **< 3.5** | ⚠️ 低评分 | ❓ 询问任务下达者 | 可能是新 Skill 或质量存疑 |

**低评分处理**:
1. 显示 Skill 基本信息（名称、作者、更新时间）
2. 说明评分情况和安全阈值
3. 等待任务下达者确认
4. 确认后进入下一步，取消则终止

### 第二层：ThreatBook 沙箱扫描

**执行流程**:
1. 临时下载 Skill 到 `/tmp/safe-install-<timestamp>`
2. 打包为 zip 文件（排除 `.git` 和 `node_modules`）
3. 上传到微步沙箱 API
4. 等待沙箱动态分析（默认 120 秒）
5. 获取分析报告并判定

**判定依据**: `summary.threat_level` 字段

| threat_level | 判定 | 操作 | 说明 |
|-------------|------|------|------|
| **clean** | ✅ safe | 允许安装 | 沙箱未检测到威胁 |
| **suspicious** | ⚠️ suspicious | ❓ 询问任务下达者 | 发现可疑行为 |
| **malicious** | ❌ malicious | 禁止安装 | 确认恶意代码 |
| **unknown** | ⚠️ suspicious | ❓ 询问任务下达者 | 无法判定，参考多引擎 |
| **API 失败** | ⚠️ api_failed | ❓ 询问任务下达者 | 网络/权限问题 |

**白名单机制**:
- 如果 `summary.is_whitelist === true`，直接判定为 safe
- 白名单文件无需进一步检查

**多引擎辅助**:
- 当 threat_level 为 unknown 时，参考 `multi_engines` 检出率
- 检出率 > 10% → 建议禁止
- 检出率 ≤ 10% → 询问任务下达者

### 决策矩阵

| 评分检查 | 沙箱扫描 | 最终决策 | 说明 |
|---------|---------|---------|------|
| ✅ ≥3.5 | ✅ safe | ✅ **直接安装** | 双重验证通过 |
| ✅ ≥3.5 | ⚠️ suspicious | ❓ **询问确认** | 高评分但内容可疑 |
| ✅ ≥3.5 | ⚠️ API 失败 | ❓ **询问确认** | 扫描服务不可用 |
| ✅ ≥3.5 | ❌ malicious | ❌ **禁止安装** | 确认恶意，无论评分 |
| ⚠️ <3.5 | ✅ safe | ❓ **询问确认** | 沙箱安全但评分低 |
| ⚠️ <3.5 | ⚠️ suspicious | ❓ **询问确认** | 双重存疑 |
| ⚠️ <3.5 | ❌ malicious | ❌ **禁止安装** | 确认恶意 |
| ⚠️ <3.5 | ⚠️ API 失败 | ❓ **询问确认** | 无法完成检查 |

## 询问任务下达者规范

### 低评分询问

```
⚠️ 此 Skill 评分低于安全阈值 (2.8/3.5)

Skill 信息:
  名称：unknown-skill
  作者：anonymous
  更新时间：2025-01-01

是否继续安装？(y/N):
```

### 可疑文件询问

```
⚠️ 沙箱扫描发现可疑内容

详细信息:
  判定：SUSPICIOUS
  可信度：70%
  引擎检出：2/8

是否继续安装？(y/N):
```

### API 失败询问

```
⚠️ 安全扫描服务暂时不可用

可能原因:
- API Key 未配置或无效
- 网络连接问题
- 服务配额用尽

是否继续安装？(y/N):
```

### 恶意文件警告（不询问，直接拒绝）

```
❌ 检测到恶意代码，禁止安装！

详细信息:
  判定：MALICIOUS
  威胁等级：malicious
  引擎检出：5/8
  威胁类型：Trojan.Generic

❌ 禁止安装恶意软件！
```

## 自动化集成

### 方式 1：Shell 函数覆盖

```bash
# 在 ~/.bashrc 中添加
clawhub() {
  if [ "$1" = "install" ] && [ -n "$2" ]; then
    node ~/.openclaw/workspace/skills/skill-safe-install/scripts/safe-install.mjs "$2" "${@:3}"
  else
    command clawhub "$@"
  fi
}
```

### 方式 2：OpenClaw 钩子

在 OpenClaw 的技能安装流程中配置前置钩子：

```javascript
// 在 OpenClaw 配置中
hooks: {
  beforeInstall: 'node ~/.openclaw/workspace/skills/skill-safe-install/scripts/safe-install.mjs'
}
```

### 方式 3：CI/CD 流水线

```yaml
# GitHub Actions 示例
- name: Safe Install Skill
  run: |
    node scripts/safe-install.mjs ${{ matrix.skill }} --dry-run
```

## 特殊情况处理

### 1. 微步沙箱 API 不可用

**处理流程**:
1. 检测 API 调用失败（超时/403/500）
2. 向任务下达者报告失败原因
3. 提供 Skill 基本信息供参考
4. 等待确认是否继续
5. 确认后跳过沙箱扫描，直接安装

### 2. 急需安装低评分 Skill

**处理流程**:
1. 报告评分情况和风险
2. 说明使用原因和必要性
3. 建议替代方案（如有）
4. 等待任务下达者确认
5. 确认后进行额外检查：
   - 查看源码
   - 检查作者其他 Skills
   - 查看更新时间
6. 使用 `--force` 安装

### 3. 大文件扫描超时

**处理流程**:
1. 默认超时 120 秒
2. 可使用 `--timeout=180` 延长
3. 如仍超时，建议：
   - 检查文件大小
   - 移除不必要的文件
   - 手动在沙箱网站上传
